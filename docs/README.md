# Vocabulary Expander Documentation

This directory is the project memory used to guide unattended implementation.

## Components

- `app/web` - React application for vocabulary exploration, visualization, and user interaction.
- `server` - API and orchestration layer for application workflows.
- `background` - long-running jobs, queues, schedulers, and asynchronous processing.
- `packages/shared` - shared TypeScript types, schemas, constants, API contracts, and utilities.
- `docs` - durable project intent, contracts, decisions, and implementation specs.
- `agent` - repo-local Codex skills and flows for repeatable project work.

## Documentation Areas

- `contracts` - current component contracts and cross-component responsibilities.
- `contracts/session-summaries` - short summaries of future project and business decisions.
- `feature-implementation` - focused implementation specs for individual features.
