# Session Summaries

This folder stores short summaries from future project sessions. They capture what was discussed, what was decided, why it was decided, and what should be revisited.

Draft summaries are review material. They become contract input only after review or after `agent/skills/contract-sync` validates them against code and accepted docs.

## File Naming

Use `YYYY-MM-DD-short-topic.md`.

## Template

```markdown
# YYYY-MM-DD - Short Topic

Status: Draft

## Context

Briefly describe what the session covered.

## Decisions

- Decision and owner/component affected.

## Rationale

- Why the decision was made.

## Open Questions

- Anything unresolved or needing review.

## Contract Follow-Up

- Contract docs that may need updates.
```
